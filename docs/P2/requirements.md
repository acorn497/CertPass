## 요구사항 분류 기준

- **기능 요구사항 (FR)**: 시스템이 수행해야 하는 기능
- **비기능 요구사항 (NFR)**: 성능, 보안, 사용성 등 품질 속성
- **우선순위**: 상 / 중 / 하

---

## 1. 기능 요구사항 (Functional Requirements)

### 1.1 인증 강화

| ID | 요구사항 | 우선순위 | 비고 |
|----|---------|---------|------|
| FR-A01 | 로그인 시 Access Token(15분)과 Refresh Token(7일)을 함께 발급한다. | 상 | Refresh Token은 HttpOnly 쿠키로 저장 |
| FR-A02 | Access Token 만료 시 Refresh Token으로 새 Access Token을 자동 재발급한다. | 상 | POST /api/v2/auth/refresh |
| FR-A03 | 로그아웃 시 서버에서 Refresh Token을 무효화한다. | 상 | DB에서 토큰 삭제 또는 블랙리스트 처리 |
| FR-A04 | Google OAuth 2.0 소셜 로그인을 지원한다. | 중 | 첫 로그인 시 자동 회원가입 |
| FR-A05 | 회원가입 시 이메일 인증 링크를 발송한다. | 중 | 미인증 계정은 로그인 가능하나 수강 신청 불가 |

## 2. 비기능 요구사항 (Non-Functional Requirements)

### 2.1 보안

| ID | 요구사항 |
|----|---------|
| NFR-A01 | Refresh Token은 HttpOnly, Secure, SameSite=Strict 쿠키로 전송한다. |
| NFR-A02 | 역할 Guard는 모든 인증 필요 엔드포인트에 적용한다. |
| NFR-A03 | 파일 업로드 시 확장자 화이트리스트와 크기 제한(5MB)을 적용한다. |
| NFR-A04 | OAuth 인증 흐름에서 CSRF 방지를 위해 state 파라미터를 사용한다. |

### 2.2 성능

| ID | 요구사항 | 목표값 |
|----|---------|-------|
| NFR-P01 | 강의 목록 전문 검색 API 응답 시간 | 평균 500ms 이하 |
| NFR-P02 | 관리자 대시보드 집계 쿼리 응답 시간 | 평균 1초 이하 |

### 2.3 사용성

| ID | 요구사항 |
|----|---------|
| NFR-U01 | 역할별 메뉴(강사/관리자)는 해당 역할 로그인 시에만 표시된다. |
| NFR-U02 | Access Token 자동 재발급이 사용자 경험에 영향을 주지 않아야 한다. (투명한 처리) |
| NFR-U03 | 강의 등록 폼에서 섹션·에피소드를 드래그 앤 드롭으로 순서 변경할 수 있다. |

## 3. 제약 사항

| 항목 | 내용 |
|------|------|
| 결제 제한 | P2에서도 무료 강의만 지원. 유료 결제는 P3에서 구현. |
| 영상 저장 | YouTube iframe 임베드 방식 유지. |
| 파일 스토리지 | 썸네일 이미지만 S3 업로드. 영상 파일 직접 업로드는 P3에서 구현. |

---

## 4. 수용 기준 (Acceptance Criteria)

| 기능 | 완료 기준 |
|------|---------|
| Refresh Token | Access Token 만료 후 자동 재발급되어 API 호출 성공 |
| 강의 등록 | 강사가 등록한 강의가 관리자 승인 후 목록에 노출 |
| 리뷰 | 수강생이 평점·리뷰 작성 후 강의 상세에 즉시 반영 |
| 모의고사 | 응시 후 점수와 오답이 결과 페이지에 정확히 표시 |
| 역할 Guard | student가 instructor 전용 API 호출 시 403 반환 |
| 검색 | 한글 시험명 검색 시 관련 강의 결과 반환 |
