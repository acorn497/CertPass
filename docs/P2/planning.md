## P2 목표

P1에서 수강생 단일 역할만 지원하던 시스템을 **역할 기반 접근 제어(RBAC)** 구조로 확장한다.
강사는 강의를 직접 등록·관리하고, 관리자는 회원·강의를 승인·관리한다.
수강생은 리뷰/평점 작성, 모의고사 응시 등 추가 학습 기능을 이용할 수 있다.

---

## 사용자 역할

| 역할 | 설명 |
|------|------|
| student (수강생) | 강의 탐색, 수강 신청, 영상 시청, 리뷰 작성, 모의고사 응시 |
| instructor (강사) | 강의 등록·수정·삭제, 내 강의 통계(수강자 수, 진도율) 조회 |
| admin (관리자) | 전체 회원 관리, 강의 승인/반려, 카테고리 관리 |

## 주요 신규 기능 (P2 범위)

### 인증 강화
- JWT Refresh Token 도입 (Access Token 15분, Refresh Token 7일)
- 소셜 로그인 (Google OAuth 2.0)
- 이메일 인증 (회원가입 시 인증 메일 발송)

### 강사 기능
- 강의 등록 폼 (제목, 설명, 섹션/에피소드, 썸네일 업로드)
- 강의 수정·삭제 (본인 강의만)
- 강사 대시보드: 수강자 수, 에피소드별 완료율, 평균 평점

### 관리자 기능
- 관리자 대시보드: 전체 회원 수, 강의 수, 수강 신청 수
- 강의 승인/반려 (isPublished → 관리자가 제어)
- 회원 역할 변경 (student ↔ instructor)
- 카테고리 추가·수정·삭제

### 수강생 추가 기능
- 강의 리뷰·평점 (1~5점, 텍스트)
- 강의 카드에 평균 평점·리뷰 수 표시
- 강의 목록에서 이미 수강 중인 강의 배지 표시
- 강의 검색 개선: 제목·강사·시험명 전문 검색 (MongoDB Text Index)
- Q&A 댓글 (수강생 질문 → 강사 답변)

### 모의고사
- 문제은행 (Questions 컬렉션): 객관식 4지선다
- 모의고사 세트 (Exams 컬렉션): 강의 연결
- 응시 기록 (ExamAttempts 컬렉션): 점수, 오답 기록
- 수강생은 수강 중인 강의의 모의고사 응시 가능

## 기술 스택 (P2 추가)

| 구분 | 기술 |
|------|------|
| 인증 | JWT (Access + Refresh Token), Google OAuth 2.0, Passport.js |
| 파일 업로드 | Multer + AWS S3 (썸네일 이미지) |
| 이메일 | Nodemailer (회원가입 인증 메일) |
| 검색 | MongoDB Text Index |
| 역할 제어 | NestJS Guard + Custom Decorator (Roles) |

---

## 비기능 요구사항 (P2)

- Access Token 만료 시 Refresh Token으로 자동 재발급 (클라이언트 인터셉터)
- 강사/관리자 전용 API는 역할 Guard로 보호
- 파일 업로드 제한: 이미지 5MB, 허용 확장자 jpg/png/webp
- 모의고사 응시 중 새로고침 시 진행 상태 유지 (sessionStorage)
