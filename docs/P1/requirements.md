## 요구사항 분류 기준

- **기능 요구사항 (FR)**: 시스템이 수행해야 하는 기능
- **비기능 요구사항 (NFR)**: 성능, 보안, 사용성 등 품질 속성
- **우선순위**: 상 / 중 / 하

---

## 1. 기능 요구사항 (Functional Requirements)

### 1.1 회원 관리

| ID | 요구사항 | 우선순위 | 비고 |
|----|---------|---------|------|
| FR-01 | 사용자는 이메일, 비밀번호, 이름을 입력하여 회원가입할 수 있다. | 상 | 이메일 중복 불가 |
| FR-02 | 사용자는 이메일과 비밀번호로 로그인할 수 있다. | 상 | JWT Access Token 발급 |
| FR-03 | 로그인한 사용자는 로그아웃할 수 있다. | 상 | 클라이언트 토큰 삭제 |
| FR-04 | 로그인한 사용자는 자신의 프로필(이름, 이메일)을 조회할 수 있다. | 중 | |
| FR-05 | 로그인한 사용자는 자신의 이름을 수정할 수 있다. | 중 | |
| FR-06 | 로그인한 사용자는 비밀번호를 변경할 수 있다. | 중 | 현재 비밀번호 확인 후 변경 |

### 1.2 강의 탐색

| ID | 요구사항 | 우선순위 | 비고 |
|----|---------|---------|------|
| FR-10 | 비회원 포함 모든 사용자는 강의 목록을 조회할 수 있다. | 상 | 페이지네이션 지원 |
| FR-11 | 사용자는 자격증 분야별로 강의 목록을 필터링할 수 있다. | 중 | 예: IT, 경제/금융, 어학, 건설/부동산 |
| FR-12 | 사용자는 강의 상세 페이지에서 강의 소개, 과목별 커리큘럼, 강사 정보를 조회할 수 있다. | 상 | |
| FR-13 | 강의 상세 페이지에서 섹션(과목)별 에피소드 목록을 확인할 수 있다. | 상 | 에피소드별 제목, 시간 표시 |

### 1.3 수강 신청

| ID | 요구사항 | 우선순위 | 비고 |
|----|---------|---------|------|
| FR-20 | 로그인한 사용자는 무료 강의를 수강 신청할 수 있다. | 상 | 중복 신청 불가 |
| FR-21 | 이미 수강 신청한 강의는 수강 신청 버튼이 비활성화된다. | 상 | |
| FR-22 | 로그인한 사용자는 자신의 수강 강의 목록(내 강의실)을 조회할 수 있다. | 상 | |

### 1.4 학습 (강의 재생)

| ID | 요구사항 | 우선순위 | 비고 |
|----|---------|---------|------|
| FR-30 | 수강 신청한 사용자는 강의 에피소드를 영상으로 시청할 수 있다. | 상 | HTML5 Video 플레이어 |
| FR-31 | 수강 신청하지 않은 사용자는 강의 영상에 접근할 수 없다. | 상 | 403 또는 리다이렉트 |
| FR-32 | 에피소드 시청 완료 시 해당 에피소드의 진도가 완료로 저장된다. | 상 | 영상 90% 이상 시청 시 완료 처리 |
| FR-33 | 강의 플레이어에서 과목(섹션) 및 에피소드 목차를 확인하고 이동할 수 있다. | 상 | |
| FR-34 | 내 강의실에서 강의별 전체 진도율(%)을 확인할 수 있다. | 중 | (완료 에피소드 수 / 전체 에피소드 수) × 100 |
| FR-35 | 마지막으로 시청한 에피소드부터 재생을 이어갈 수 있다. | 중 | |

## 2. 비기능 요구사항 (Non-Functional Requirements)

### 2.1 성능

| ID | 요구사항 | 목표값 |
|----|---------|-------|
| NFR-01 | 강의 목록 API 응답 시간 | 평균 300ms 이하 |
| NFR-02 | 로그인 API 응답 시간 | 평균 500ms 이하 |
| NFR-03 | 영상 로딩 시작 시간 | 3초 이내 |

### 2.2 보안

| ID | 요구사항 |
|----|---------|
| NFR-10 | 비밀번호는 bcrypt(salt rounds ≥ 10)로 해싱하여 저장한다. |
| NFR-11 | JWT는 Access Token만 사용하며, 만료 시간은 1시간으로 설정한다. |
| NFR-12 | 인증이 필요한 API는 Authorization 헤더의 Bearer 토큰을 검증한다. |
| NFR-13 | 민감 정보(DB URI, JWT Secret 등)는 환경 변수로 관리한다. |
| NFR-14 | CORS는 허용 오리진을 명시적으로 설정한다. |

### 2.3 사용성

| ID | 요구사항 |
|----|---------|
| NFR-20 | 모바일(320px 이상) ~ 데스크탑(1280px 이상) 반응형 레이아웃을 지원한다. |
| NFR-21 | 폼 입력 오류 시 사용자에게 명확한 오류 메시지를 표시한다. |
| NFR-22 | 로딩 중에는 스피너 또는 스켈레톤 UI를 표시한다. |

### 2.4 유지보수성

| ID | 요구사항 |
|----|---------|
| NFR-30 | 백엔드는 NestJS 모듈 단위로 관심사를 분리한다. |
| NFR-31 | API 응답은 일관된 JSON 구조({ success, data } / { success, message })를 사용한다. |
| NFR-32 | TypeScript를 사용하여 타입 안전성을 보장한다. |

## 3. 제약 사항

| 항목 | 내용 |
|------|------|
| P1 역할 제한 | P1에서는 수험생 역할만 지원. 강사·관리자 기능은 P2에서 구현. |
| 결제 제한 | P1에서는 무료 강의만 지원. 유료 결제는 P3에서 구현. |
| 영상 저장 | 영상은 YouTube iframe 임베드 방식 사용. videoUrl 필드에 YouTube 영상 ID 저장. |
| 모의고사 | 기출문제/모의고사 풀이 기능은 P2에서 구현. |

## 4. 수용 수준 (Acceptance Criteria) 요약

| 기능 | 완료 기준 |
|------|---------|
| 회원가입 | 유효한 이메일·비밀번호로 가입 후 로그인 가능 |
| 로그인 | 토큰 발급 후 인증이 필요한 API 정상 접근 |
| 강의 목록 | 목록 조회 및 자격증 분야 필터 동작 확인 |
| 강의 상세 | 과목별 섹션/에피소드 목록 정상 표시 |
| 수강 신청 | 중복 신청 방지 및 내 강의실 반영 확인 |
| 강의 재생 | 영상 재생 및 시청 완료 시 진도 저장 확인 |
| 진도율 | 내 강의실에서 진도율 % 정상 표시 |
