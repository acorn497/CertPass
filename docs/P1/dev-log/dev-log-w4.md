## 2. 단계 별 작업 내용

### Week 6 (4/2~4/5) — 최종 점검 준비

**작업 방향 정리**
- 기존 MVP 기능(인증, 강의 목록/상세, 플레이어, 내 강의실, 마이페이지)의 동작을 기준으로 오류 재현 케이스를 정리
- 제출 전 우선순위를 `기능 안정성 > 데이터 정합성 > 문서화`로 설정

**검증 포인트**
- 커리큘럼 응답 구조와 프론트 파싱 로직 일치 여부
- 내 강의실 진도 데이터 계산 및 표시 일관성
- 플레이어 재생 옵션(자동재생/종료 처리/이동 동작) 안정성

---

### Week 6 (4/6) — 핵심 버그 수정

**주요 변경 사항**
- `courses.service.ts`: 커리큘럼 조회 로직 보완으로 강의 상세/구성 데이터 반환 안정화
- `enrollments.service.ts`: 내 강의실 데이터 파싱 및 응답 형태 보정
- `PlayerPage.tsx`: 플레이어 영상 재생 설정 보완으로 시청 흐름 개선
- `seed.ts`: 샘플 데이터 정리 및 테스트 데이터 신뢰도 향상

**결과**
- 강의 상세 및 내 강의실에서 데이터 파싱 오류로 인한 화면 깨짐 케이스 해소
- 플레이어 진입 후 재생 동작이 더 일관되게 동작하도록 개선

---

### Week 6 (4/9) — 최종 결과 통합 및 문서화

**프론트엔드 정리**
- 주요 페이지(`Home`, `Courses`, `CourseDetail`, `Player`, `MyCourses`, `MyPage`, `Login`, `Register`) 전반 구조/상태 처리 정리
- 공통 컴포넌트(`Header`, `Footer`, `CourseCard`) 스타일 및 표시 로직 보완
- API 연동 레이어(`api/courses.ts`) 호출 흐름 및 응답 처리 정리

**백엔드 정리**
- 인증/강의/에피소드/수강/진도/유저 컨트롤러 전반 응답 처리 보정
- `courses.service.ts`, `enrollments.service.ts`, `progress.service.ts` 중심으로 데이터 가공 로직 정리
- `seed.ts` 재정비로 데모 및 검증용 데이터 구성 개선

**산출물 정리**
- 발표 자료 작성 및 PDF 생성 완료
- 개발 문서(`dev-log`, `dev-plan`, `todo`) 최신화
