# 요구사항 명세서 — 가정용 소모품 재고 관리 앱

## 1. 개요

가정에서 사용하는 소모품(칫솔, 치약, 우유, 휴지 등)의 재고를 관리하는 모바일 앱이다. 서버 통신 없이 로컬 데이터베이스만으로 동작하며, 바코드 스캔과 수동 입력 두 가지 방식으로 아이템을 관리한다.

## 2. 사용자 시나리오

### 2.1 바코드가 있는 공산품 등록
1. 사용자가 "아이템 추가" 버튼을 누른다.
2. 바코드 스캔을 선택한다.
3. 카메라로 바코드를 촬영한다.
4. 바코드 번호가 자동으로 입력된다.
5. 이름, 수량, 사용기한(선택), 저장 위치(선택)를 입력한다.
6. 저장 버튼을 누른다.

### 2.2 바코드 없는 아이템 등록 (농산물 등)
1. 사용자가 "아이템 추가" 버튼을 누른다.
2. 수동 입력을 선택한다.
3. 이름, 수량, 사용기한(선택), 저장 위치(선택)를 입력한다.
4. 저장 버튼을 누른다.

### 2.3 재고 수량 수정
1. 홈 화면에서 아이템을 검색한다.
   - 바코드 스캔으로 검색하거나
   - 이름으로 텍스트 검색한다.
2. 검색 결과에서 아이템을 선택한다.
3. 수량을 수정한다 (직접 입력 또는 +/- 버튼).
4. 저장 버튼을 누른다.

### 2.4 아이템 삭제
1. 아이템 상세 화면에서 삭제 버튼을 누른다.
2. 삭제 확인 다이얼로그가 표시된다.
3. 확인 시 아이템이 삭제된다.

## 3. 기능 요구사항

### 3.1 아이템 관리

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-ITEM-01 | 바코드 촬영을 통해 아이템을 등록할 수 있다 | O |
| F-ITEM-02 | 수동으로 이름을 입력하여 아이템을 등록할 수 있다 | O |
| F-ITEM-03 | 기존 아이템의 정보를 수정할 수 있다 | O |
| F-ITEM-04 | 아이템을 삭제할 수 있다 | O |
| F-ITEM-05 | 동일 바코드 스캔 시 기존 상품을 찾아 새 재고 항목을 추가할 수 있다 | O |

### 3.2 데이터 필드

#### 상품 (Product)

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | int (auto) | O | 내부 고유 식별자 |
| barcode | String | X | 바코드 번호 (공산품, UNIQUE) |
| name | String | O | 상품 이름 |
| createdAt | DateTime | O | 등록일 |
| updatedAt | DateTime | O | 최종 수정일 |

#### 재고 항목 (InventoryItem)

같은 상품이라도 구매 시기·구매처에 따라 사용기한과 메모가 다를 수 있으므로 상품과 분리하여 관리한다.

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | int (auto) | O | 내부 고유 식별자 |
| productId | int (FK) | O | 소속 상품 |
| quantity | int | O | 현재 수량 (기본값: 1) |
| expirationDate | DateTime? | X | 사용기한 (null이면 무기한) |
| memo | String | X | 메모 (자유 텍스트, 구매처·구매일 등) |
| storageLocation | String | X | 저장 위치 (예: 냉장고, 욕실 선반, 창고) |
| createdAt | DateTime | O | 등록일 |
| updatedAt | DateTime | O | 최종 수정일 |

### 3.3 검색

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-SEARCH-01 | 바코드 촬영으로 아이템을 검색할 수 있다 | O |
| F-SEARCH-02 | 이름 텍스트로 아이템을 검색할 수 있다 (부분 일치) | O |
| F-SEARCH-03 | 저장 위치별로 아이템을 필터링할 수 있다 | X |

### 3.4 재고 수량 관리

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-QTY-01 | +/- 버튼으로 수량을 조절할 수 있다 | O |
| F-QTY-02 | 수량을 직접 숫자로 입력할 수 있다 | O |
| F-QTY-03 | 수량이 0 이하가 되지 않도록 한다 | O |

### 3.5 저장 위치

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-LOC-01 | 아이템 등록/수정 시 저장 위치를 입력할 수 있다 | O |
| F-LOC-02 | 이전에 입력한 저장 위치를 자동완성으로 제안한다 | X |

### 3.6 사용기한 알림

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-EXP-01 | 사용기한을 날짜 또는 무기한으로 설정할 수 있다 | O |
| F-EXP-02 | 사용기한 임박 아이템을 홈 화면 상단에 별도 섹션으로 표시한다 | O |
| F-EXP-03 | 사용기한이 지난 아이템을 시각적으로 경고 표시한다 | O |
| F-EXP-04 | 임박 기준일을 설정할 수 있다 (기본값: 3일) | X |
| F-EXP-05 | 무기한 아이템은 사용기한 알림 대상에서 제외한다 | O |

### 3.7 메모

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-MEMO-01 | 아이템 등록/수정 시 자유 텍스트 메모를 입력할 수 있다 | O |
| F-MEMO-02 | 아이템 상세 화면에서 메모를 확인할 수 있다 | O |

### 3.8 바코드 스캔

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-SCAN-01 | 카메라를 이용해 1D/2D 바코드를 스캔할 수 있다 | O |
| F-SCAN-02 | 스캔 결과를 바코드 필드에 자동 입력한다 | O |
| F-SCAN-03 | 카메라 권한이 없을 때 권한 요청 안내를 표시한다 | O |

## 4. 비기능 요구사항

| ID | 요구사항 |
|----|---------|
| NF-01 | 모든 데이터는 로컬 데이터베이스에 저장한다 (서버 통신 없음) |
| NF-02 | 오프라인 환경에서 모든 기능이 동작해야 한다 |
| NF-03 | Android, iOS 최소 지원 (데스크톱/웹은 바코드 스캔 미지원 가능) |
| NF-04 | 한국어 UI를 기본으로 한다 |

## 5. 화면 구성

### 5.1 홈 화면 (아이템 목록)
- 등록된 아이템 목록 표시 (이름, 수량, 사용기한, 저장 위치)
- 상단 검색바 (텍스트 검색)
- 바코드 스캔 검색 버튼
- 아이템 추가 FAB 버튼
- 사용기한 임박/만료 아이템 별도 섹션 (상단에 경고 표시)

### 5.2 아이템 추가/수정 화면
- 바코드 스캔 버튼 (등록 시)
- 바코드 번호 표시 필드 (읽기 전용 또는 수동 입력)
- 이름 입력 필드
- 수량 입력 (+/- 버튼 및 직접 입력)
- 사용기한 선택 (DatePicker 또는 "무기한" 토글)
- 메모 입력 필드 (여러 줄 텍스트)
- 저장 위치 입력 (텍스트 + 자동완성)
- 저장 / 취소 버튼

### 5.3 아이템 상세 화면
- 아이템 전체 정보 표시 (메모 포함)
- 수량 빠른 조절 (+/- 버튼)
- 수정 버튼 → 수정 화면으로 이동
- 삭제 버튼

## 6. 제약사항
- 바코드 스캔은 카메라가 있는 기기(모바일)에서만 동작한다.
- 데스크톱/웹 환경에서는 바코드 번호 수동 입력으로 대체한다.
- 바코드에서 상품명을 자동으로 가져오는 외부 API 연동은 범위 밖이다 (로컬 전용).
