# 테스트 계획

## 실행 환경

- `flutter test` — 단위 테스트, 위젯 테스트
- `flutter analyze` — 정적 분석 (매 Phase 완료 시)
- 바코드 스캔 등 하드웨어 의존 기능은 mock 처리하여 테스트

## 테스트 디렉토리 구조

```
test/
├── models/               # 모델 단위 테스트
├── db/                   # DAO 단위 테스트 (sqflite_common_ffi 사용)
├── providers/            # 상태관리 로직 테스트
├── widgets/              # 공통 위젯 테스트
└── screens/              # 화면 위젯 테스트
```
