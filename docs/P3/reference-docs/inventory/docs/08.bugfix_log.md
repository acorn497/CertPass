## BUG-001: 재고 수정 화면에서 Back 시 상세 화면 Back 버튼 소실 및 앱 종료

### 발생일
2026-03-16

### 증상
1. 홈 → 상품 상세 → 재고 수정 화면 진입
2. 재고 수정 화면에서 Back (뒤로가기)
3. 상품 상세 화면으로 돌아오지만 **AppBar의 Back 버튼이 사라짐**
4. 이 상태에서 물리 Back 버튼 누르면 **홈이 아닌 앱이 종료됨**

### 원인
go_router에서 `context.go()`와 `context.push()`의 동작 차이:
- `context.go()` — 네비게이션 스택을 **교체** (기존 스택 제거)
- `context.push()` — 네비게이션 스택 위에 **추가** (기존 스택 유지)

두 곳에서 `context.go()`를 사용하여 스택이 교체되었다:
1. `lib/screens/product/product_detail_screen.dart:130` — 재고 추가 버튼
2. `lib/screens/product/widgets/inventory_item_card.dart:42` — 재고 수정 탭

`go()`로 재고 수정 화면에 진입하면 상세 화면이 스택에서 제거된다. 재고 수정 화면에서 `pop()` 시 스택에 상세 화면이 없으므로 go_router가 루트(`/`)로 fallback하지만, 이 때 홈 화면에는 이전 네비게이션 이력이 없어 Back 버튼이 없고 물리 Back으로 앱이 종료된다.

### 수정 내용

**파일 1: `lib/screens/product/product_detail_screen.dart`**
```dart
// 변경 전
context.go(AppRoutes.inventoryNew(productId.toString()));

// 변경 후
context.push(AppRoutes.inventoryNew(productId.toString()));
```

**파일 2: `lib/screens/product/widgets/inventory_item_card.dart`**
```dart
// 변경 전
context.go(AppRoutes.inventoryEdit(productId.toString(), item.id.toString()));

// 변경 후
context.push(AppRoutes.inventoryEdit(productId.toString(), item.id.toString()));
```

### go_router 네비게이션 규칙 정리

| 메서드 | 동작 | 사용 시점 |
|--------|------|----------|
| `context.go('/')` | 스택 교체, 해당 경로로 이동 | 홈으로 돌아갈 때 (스택 초기화 의도) |
| `context.push('/path')` | 스택 위에 추가 | 하위 화면으로 진입할 때 (Back 지원 필요) |
| `context.pop()` | 스택에서 제거, 이전 화면으로 | 현재 화면 닫기 |

### 검증
- `flutter analyze` — 0건
- 홈 → 상세 → 재고 수정 → Back → 상세(Back 버튼 정상) → Back → 홈 정상 동작 확인
