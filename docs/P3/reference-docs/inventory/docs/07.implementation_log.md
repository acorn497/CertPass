## Phase 2. 데이터베이스

### 생성된 파일
- `lib/db/tables/products_table.dart` — products 테이블 (barcode UNIQUE nullable)
- `lib/db/tables/inventory_items_table.dart` — inventory_items 테이블 (FK CASCADE, quantity CHECK >= 0)
- `lib/db/tables/storage_locations_table.dart` — storage_locations 테이블 (name UNIQUE)
- `lib/db/tables/app_settings_table.dart` — app_settings 테이블 (key PK)
- `lib/db/app_database.dart` — `@DriftDatabase`, FK 활성화, Seed 데이터 삽입
- `lib/db/daos/product_dao.dart` — ProductDao (CRUD, 바코드/이름 검색, 총 수량 집계 JOIN)
- `lib/db/daos/inventory_item_dao.dart` — InventoryItemDao (CRUD, 임박/만료 조회)
- `lib/db/daos/storage_location_dao.dart` — StorageLocationDao
- `lib/db/daos/app_setting_dao.dart` — AppSettingDao (upsert)
- `lib/db/app_database.g.dart` — drift 자동 생성 코드

### 설계 결정
- drift가 자체 데이터 클래스(`Product`, `InventoryItem` 등)를 생성하므로 별도 freezed 모델은 불필요
- `AppDatabase.forTesting(NativeDatabase.memory())` 생성자로 테스트 시 in-memory DB 사용
- `inventory_items.quantity`의 CHECK 제약은 `customConstraint` 사용 (drift의 `.check()` 메서드 재귀 문제 회피)
- 날짜는 drift의 `dateTime()` 타입 사용 (내부적으로 ISO 8601 TEXT 저장)

## Phase 3. 상태관리 및 비즈니스 로직

### 생성된 파일
- `lib/providers/database_provider.dart` — AppDatabase 싱글톤 Provider + 각 DAO Provider
- `lib/providers/product_provider.dart` — 상품 목록(총 수량 포함), 단건 조회, 바코드/이름 검색, CRUD 액션
- `lib/providers/inventory_provider.dart` — 상품별 재고, 임박/만료 필터, 수량 증감, CRUD 액션
- `lib/providers/storage_location_provider.dart` — 위치 목록, 추가/삭제 액션
- `lib/providers/app_setting_provider.dart` — 사용기한 임박 기준일, 설정 읽기/쓰기 액션

### 설계 결정
- `@riverpod` 코드 생성 대신 수동 Provider 정의 사용 — build_runner 단계 불필요, 일관성 유지
- Stream 기반 Provider로 drift의 `watch()` 쿼리 활용 — DB 변경 시 UI 자동 갱신
- 액션 클래스(ProductActions, InventoryActions 등)는 mutation 후 관련 Provider를 invalidate하여 UI 일관성 보장
- `expiringItemsProvider`가 `expirationAlertDaysProvider`에 의존 — 기준일 변경 시 자동 갱신

## Phase 4. 공통 위젯

### 생성된 파일
- `lib/widgets/quantity_stepper.dart` — pill 스타일, −/+/직접입력, compact 모드(32x32)
- `lib/widgets/expiry_date_picker.dart` — DatePicker + 무기한 토글 Switch
- `lib/widgets/storage_location_picker.dart` — ConsumerWidget, DropdownButtonFormField
- `lib/widgets/delete_confirm_dialog.dart` — static `show()` 메서드, error 색상 삭제 버튼
- `lib/widgets/search_bar_widget.dart` — pill 검색바 + 원형 바코드 스캔 버튼

### 설계 결정
- StorageLocationPicker는 `ConsumerWidget`으로 storageLocationsProvider를 직접 watch — 외부에서 데이터 주입 불필요
- QuantityStepper의 compact 모드는 상세 화면 재고 카드의 빠른 수량 조절에 사용
- 모든 위젯 텍스트는 `AppLocalizations` 사용 — 하드코딩 문자열 없음
